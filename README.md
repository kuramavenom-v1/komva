# كومڤا (Komva) — منصة محادثات عربية

نسخة MVP حقيقية شغّالة: تسجيل دخول (بريد أو هاتف)، محادثات فردية لحظية،
بحث عن مستخدمين، تعديل الملف الشخصي. مبنية بـ Next.js 14 + Supabase.

---

## 1) قبل ما تبدأ — المتطلبات

- Node.js 18 أو أحدث مثبت على جهازك ([nodejs.org](https://nodejs.org))
- حساب Supabase (عندك مسبقاً ✓)
- محرر أكواد مثل VS Code

---

## 2) تثبيت المشروع على جهازك

افتح Terminal داخل مجلد `komvaWeb` ونفّذ:

```bash
npm install
```

هذا الأمر يحتاج إنترنت (يحمّل الحزم من npm)، شي ما كان متاح في بيئة التطوير
اللي جهزت فيها المشروع، فلازم تشغّله أول مرة على جهازك أنت.

---

## 3) ربط المشروع بـ Supabase

### أ) إنشاء قاعدة البيانات

1. ادخل [supabase.com/dashboard](https://supabase.com/dashboard) → افتح مشروعك
2. من القائمة الجانبية: **SQL Editor** → **New query**
3. افتح ملف `supabase/schema.sql` الموجود في هذا المشروع، انسخ محتواه كاملاً،
   الصقه في المحرر، واضغط **Run**
4. تأكد إنه نفّذ بدون أخطاء (راح ينشئ 4 جداول: profiles, chats,
   chat_participants, messages + الصلاحيات الأمنية + تفعيل الـ Realtime)

### ب) تفعيل تسجيل الدخول بالهاتف (OTP)

⚠️ **هذي الخطوة تحتاج حساب Twilio مدفوع** (تكلفة بسيطة لكل رسالة SMS، مفيش
حل مجاني حقيقي لهذا الجزء):

1. أنشئ حساب على [twilio.com](https://twilio.com) واحصل على:
   - Account SID
   - Auth Token
   - رقم مرسل (Messaging Service SID أو رقم هاتف Twilio)
2. في Supabase Dashboard: **Authentication** → **Providers** → **Phone**
3. فعّل Phone provider واختر Twilio، وأدخل المفاتيح الثلاثة
4. احفظ

إذا ما تبي تفعّل هذا الآن، التطبيق يشتغل تمام بدون مشاكل — فقط خانة "رقم
الهاتف" في صفحة التسجيل ما راح تشتغل لين تفعّلها.

### ج) نسخ المفاتيح إلى المشروع

1. في Supabase Dashboard: **Project Settings** (⚙️) → **API**
2. انسخ:
   - **Project URL**
   - **anon public key**
3. في مجلد المشروع، انسخ `.env.local.example` وسمّه `.env.local`:

```bash
cp .env.local.example .env.local
```

4. افتح `.env.local` وحط القيم الحقيقية:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
```

---

## 4) تشغيل المشروع محلياً

```bash
npm run dev
```

افتح المتصفح على: **http://localhost:3000**

راح يحولك تلقائياً لصفحة تسجيل الدخول. اضغط "إنشاء حساب جديد" وجرّب.

---

## 5) كيف تجرب المحادثات اللحظية

عشان تشوف الرسائل تنزل لحظياً بين شخصين:

1. افتح المتصفح في نافذتين (أو نافذة عادية + نافذة متخفية)
2. سجّل حساب مختلف في كل نافذة
3. من إحدى النافذتين، اضغط أيقونة القلم (محادثة جديدة) وابحث عن اسم
   المستخدم التاني
4. ابدأ المحادثة وجرّب ترسل من الطرفين — الرسائل تظهر فوراً بدون تحديث
   الصفحة

## 6) هيكلة المشروع

```
komvaWeb/
├── app/
│   ├── login/              صفحة تسجيل الدخول
│   ├── register/           صفحة إنشاء حساب
│   └── chat/
│       ├── layout.tsx      يجلب بيانات المستخدم والمحادثات
│       ├── page.tsx        الحالة الفارغة (بدون محادثة مختارة)
│       ├── settings/       تعديل الملف الشخصي
│       └── [chatId]/       صفحة محادثة محددة
├── components/
│   ├── AuthForm.tsx         نموذج الدخول/التسجيل (بريد + هاتف)
│   └── chat/
│       ├── Sidebar.tsx          قائمة المحادثات
│       ├── ChatWindow.tsx       نافذة المحادثة + الإرسال اللحظي
│       ├── NewChatModal.tsx     البحث عن مستخدم وبدء محادثة
│       ├── MessageBubble.tsx    فقاعة الرسالة
│       └── Avatar.tsx           الصورة الرمزية
├── lib/supabase/
│   ├── client.ts            عميل Supabase للمتصفح
│   ├── server.ts             عميل Supabase للخادم
│   ├── middleware.ts         حماية الصفحات + تحديث الجلسة
│   └── database.types.ts     أنواع TypeScript لقاعدة البيانات
└── supabase/
    └── schema.sql             سكريبت إنشاء قاعدة البيانات كاملاً
```

## 7) إيش الميزات الموجودة فعلياً (شغالة 100%)

- ✅ تسجيل دخول بالبريد الإلكتروني وكلمة مرور
- ✅ تسجيل دخول برقم الهاتف (يحتاج تفعيل Twilio — راجع القسم 3ب)
- ✅ إنشاء حساب جديد + ملف شخصي تلقائي
- ✅ تعديل اسم المستخدم والنبذة التعريفية
- ✅ البحث عن مستخدمين آخرين بالاسم
- ✅ بدء محادثة فردية جديدة
- ✅ إرسال واستقبال رسائل **لحظياً** (Realtime)
- ✅ حماية البيانات: كل مستخدم يشوف فقط محادثاته (Row Level Security)
- ✅ حماية الصفحات: ما تقدر تدخل `/chat` بدون تسجيل دخول

## 8) إيش الناقص (المراحل القادمة حسب الخطة)

- ⏳ المجموعات (الجدول جاهز في الخطة لكن الواجهة غير مبنية بعد)
- ⏳ المكالمات الصوتية والفيديو (أزرارها موجودة بالواجهة لكن معطّلة)
- ⏳ إرسال الصور والملفات
- ⏳ حالة "يكتب الآن" وحالة القراءة
- ⏳ حسابات الأعمال والاشتراكات
- ⏳ التشفير الطرفي الكامل

---

## 9) النشر (Deploy) مجاناً

أسهل طريقة:

1. ارفع المشروع على GitHub
2. ادخل [vercel.com](https://vercel.com) → New Project → اختر الـ repo
3. في إعدادات Environment Variables، ضيف نفس المتغيرين من `.env.local`
4. Deploy

---

## 10) لو واجهت مشكلة

- **صفحة بيضاء أو خطأ Supabase**: تأكد إن `.env.local` فيه القيم الصحيحة
  وإنك أعدت تشغيل `npm run dev` بعد تعديله
- **الرسائل ما توصل لحظياً**: تأكد إنك نفّذت آخر سطرين من `schema.sql`
  (تفعيل Realtime) — لو فاتتك، روح SQL Editor ونفّذها بمفردها:
  ```sql
  alter publication supabase_realtime add table public.messages;
  ```
- **خطأ "relation does not exist"**: معناته `schema.sql` ما اتنفذ بالكامل،
  ارجع للقسم 3أ ونفّذه من جديد
